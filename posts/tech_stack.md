On the other end of this experience and as the project grew I now have a greater appreciation for frameworks!


Complexity grows quickly and it’s easy to loose track of which part does what in one giant html file.


## Tech Stack
**Pure HTML, CSS and Javascript**.


>I didn’t use a framework because I was focusing on rapid iteration
>wasn’t a full-fledged web developer when I started the project,
>I just started and kept going..


### IDE
I used **VS Code** as my IDE, its vast extension library is invaluable to my development process.

Un-ironically, **Ctrl+F** find and replace has been instrumental in navigating around my codebase. 
*I have since learned that a framework packages html & css snipets together which would have made my life much easier as the project grew.*

### Browser-based development
This whole web development journey wouldn’t have happened without **Liver Server** extension.
*When I started I didn't know how to use Git, so I developed iteratively locally on my pc.*
Live Server enabled me to host my site in my browser, which let me iterate over design, UI/UX, and mobile optimisation.

I used **Firefox**, it’s been my ‘daily driver’ browser for years now. 

Using the **developer tools** and inspecting page elements helped me troubleshoot and hotfix my css, looking up snippets with Ctrl+F.

**Mobile Simulator** by **Francois**, a Firefox extension that provides multiple device skins and reduced window sizes. 
I toggled Mobile Simulator to test how my site worked and looked on desktop, phones,  tablet and laptop screens. 
I think would be a great tool if you wanted to create SaaS marketing content.

## Review, Testing and Compliance
I used **Google Lighthouse**, a free tool nested within the **Google Chrome browser dev tools** to evaluate my sites load time performance, accessibility and to identify poor implementation of features.
